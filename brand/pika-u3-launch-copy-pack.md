## 5) CTA Hierarchy

### Primary CTA Set
Use when the product or waitlist is the main action.

- **Build your room**
- **Join the waitlist**

### Secondary CTA Set
Use for lower-emphasis support actions.

- **See how it works**
- **Learn more**
- **Share Pika**
- **Invite your people**

### CTA Rules
- Prefer **Build your room** anywhere the user can start the core experience
- Prefer **Join the waitlist** anywhere access is gated
- Use **See how it works** as the main secondary action on launch surfaces
- Use **Invite your people** and **Share Pika** only on referral or post-signup share surfaces
- Do not pair the alternate support hook with a CTA that changes the primary action hierarchy

## 7) Usage Rules

### Required
Use **Build your room. Rank with friends.** exactly as written on:
- launch hero
- primary launch cards
- headline-first landing assets
- top-level email headers for launch
- any asset representing the main product promise in a single line

### Allowed as Alternate Support
Use **Stop comparing in circles. Rank it with Pika.** only:
- as an alternate headline in tests
- on secondary share assets
- on supporting carousel frames
- in below-the-fold modules where the canonical hook has already led
- in internal variation slots where a secondary option is needed

### Not Allowed
- Do not replace the primary launch hero with the alternate support hook
- Do not present the alternate support hook as the canonical line
- Do not introduce new taglines or new positioning claims in this pack
- Do not swap CTA hierarchy just because the alternate support hook is used

---

## 8) Tone Guardrails for These Assets

Keep the copy:
- warm
- lightly witty
- specific
- social
- useful

Avoid:
- startup clichés
- generic productivity language
- empty community phrasing
- sounding like a poll tool
- sounding like enterprise software pretending to be fun

Preferred texture:
- real choices
- real friends
- messy opinions becoming clearer
- group chat chaos turning into an actual ranking
