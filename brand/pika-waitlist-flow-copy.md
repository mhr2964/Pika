**Voice target:** warm, witty, compact, specific  
**Avoid:** corporate filler, hype, exclamation overload, “journey,” “unlock,” “seamless,” “revolutionary”

## 15) Referral Fraud / Abuse Guardrails

**Those referrals don’t look valid yet.**

### Tone note
Keep this calm and factual. Never accuse the user of cheating directly.

## 21) Tone Guardrails for Implementation

Use this copy as-is unless space is extremely tight. If trimming is needed:

- Keep sentences short
- Prefer plain words over startup slang
- Stay lightly funny, never smug
- Be reassuring in error states
- Avoid sounding exclusive in a try-hard way
- Do not use exclamation marks unless there is a true product reason

## 23) Recommended Default Sequence

1. User sees landing headline, subhead, email field, and primary CTA.
2. On submit success, show confirmation plus referral CTA immediately.
3. If referral tracking exists, show current count and next milestone.
4. If user returns with an existing email, switch to already-joined state instead of implying a fresh signup.
5. Keep all error messages inline, human, and specific.
