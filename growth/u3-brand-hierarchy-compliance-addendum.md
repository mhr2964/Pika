# U3 Brand Hierarchy Compliance Addendum

This addendum applies to rollout using:
- `u3-messaging-lock.md`
- `u3-acquisition-pack.md`
- `u3-message-lock-application-map.md`
- `u3-acquisition-asset-implementation-checklist.md`
- `workspace/brand/pika-u3-launch-copy-pack.md`

## 1) Fixed Primary Hook Rule

On all launch-critical acquisition and referral surfaces, the primary hook remains:

**Build your room. Rank with friends.**

This line must be the first and most prominent message on:
- acquisition landing surfaces
- waitlist signup and confirmation surfaces
- referral invite surfaces
- share-loop surfaces where hierarchy can be preserved

Do not replace, rotate, or test against this line on primary rollout surfaces.

---

## 2) Subordinate-Only Rule for Alternate Lines

Any alternate brand line may appear only as subordinate support.

Allowed:
- subhead
- body copy
- lower-priority reinforcement line
- secondary test treatment that does not displace the primary hook

Not allowed:
- headline replacement
- pre-head above the canonical hook
- top-line share message replacement
- waitlist or referral module title replacement

If an alternate line competes with the primary hook for first read, it is out of compliance.

---

## 3) Waitlist-Movement Language Rule

Waitlist-movement language appears only on invite-enabled surfaces.

Allowed examples:
- referral invite screen
- waitlist confirmation state with invite action
- referral explainer block
- share-loop surfaces that explicitly include invite/join mechanics

Not allowed:
- generic landing hero
- non-invite acquisition placements
- headline-level messaging
- surfaces without a visible invite or join action

Use waitlist movement as a referral mechanic explanation, not as standalone positioning.

---

## 4) Compressed Placement Rule

When format constraints require compression, use:

**Canonical hook + one approved supporting line only**

Approved supporting-line types:
- Join the waitlist.
- Join my room.
- Compare picks.
- See where everyone lands.
- Invite your room.

Compression rule:
- keep **Build your room. Rank with friends.** first
- add only one approved supporting line
- trim all other copy before changing hierarchy
- if the hook cannot remain first, the placement is non-compliant for rollout use

---

## 5) Surface Allowance Table

| Surface Type | Primary Hook Required | Alternate Line Allowed | Waitlist-Movement Language Allowed | Compressed Rule |
|---|---|---|---|---|
| Acquisition | yes | subordinate only | no, unless invite is present | hook + one approved supporting line |
| Waitlist | yes | subordinate only | only when invite action is present | hook + one approved supporting line |
| Referral | yes | subordinate only | yes | hook + one approved supporting line |
| Share-loop | yes where hierarchy can be preserved | subordinate only | only when invite/join action is explicit | hook + one approved supporting line |

## Operational Use
If a placement fails any rule above, treat it as a hierarchy exception and revise the surface before rollout.
