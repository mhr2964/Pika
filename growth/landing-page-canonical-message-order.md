# Pika landing-page canonical message order
_Status: approved single-route selection for live acquisition package_

## Selected primary hero / subhead / CTA hierarchy

### Hero
**Headline**  
Stop losing the plan in the chat.

**Subhead**  
Pika turns messy group messages into one shareable plan everyone can see and react to.

**Primary CTA**  
Get early access

**Secondary CTA**  
None on initial hero state.

### Selection rationale
This is the strongest single route because it:
- leads with the most recognizable pain
- names the native behavior/habitat: chat
- lands the product outcome in simple terms: one shareable plan
- stays inside the approved launch-support narrative
- avoids over-claiming breadth or drifting into generic planning language

## CTA hierarchy rules
- Primary CTA label is always: **Get early access**
- No competing CTA above the fold
- No “book demo,” “learn more,” or “start free” language
- Repeat the same CTA after benefit explanation and in the final block
- If a footer CTA is present, it should also remain **Get early access**

---

## Messaging hierarchy summary
1. Stop losing the plan in the chat.
2. Pika turns messy group messages into one shareable plan everyone can see and react to.
3. Group chat planning gets buried and repeated.
4. Pika gives the group one clear shared plan.
5. It works for real social planning use cases.
